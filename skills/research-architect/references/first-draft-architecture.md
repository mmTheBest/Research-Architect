# First-Draft Architecture

The first draft should be readable, but its main purpose is structural correctness. It is better to produce a rough but logically auditable draft than polished prose with unsupported claims.

## Flexible section model

Do not force every project into one template. Choose a structure that fits the research spine and evidence mode.

Common section functions:

| Function | Purpose |
|---|---|
| Title | Names object, approach, and bounded contribution |
| Abstract/Summary | Compresses the research spine |
| Introduction | Builds motivation, gap, and contribution |
| Background/Related Work | Defines source clusters and positioning |
| Methods/Study Design | Explains how evidence was generated |
| Results/Analysis | Presents evidence in claim-support order |
| Discussion | Interprets evidence and relates to prior work |
| Limitations | States scope and failure modes |
| Conclusion | Restates the supported contribution only |

## Section blueprint schema

| Section | Function | Key Claim | Evidence Needed | Citation Support | Figures/Tables | Risk | Drafting Instruction |
|---|---|---|---|---|---|---|---|

## Writing rationale matrix schema

| Row ID | Manuscript Unit | Planned Function | Research-Spine Link | Literature/Exemplar Pattern Learned | Evidence or Citation Anchor | Planned Text Move | Claim Boundary | Final Check |
|---|---|---|---|---|---|---|---|---|

## Drafting rule

Write each unit only after its row in the rationale matrix is clear. If a row cannot identify evidence or citation support, the text must become background, limitation, future work, or be removed.
