# Literature and Exemplar Mapping Framework

## Literature review is not summarization

The objective is to turn sources into a map of concepts, methods, evidence standards, and gaps.

## Source roles

| Role | Purpose | Typical Use in Paper |
|---|---|---|
| Concept source | Defines the phenomenon, construct, or task | Introduction/background |
| Method source | Defines technical approach or analysis method | Methods/related work |
| Baseline source | Provides comparison target | Results/evaluation |
| Benchmark/source-data paper | Establishes dataset, protocol, or measurement norm | Methods/results |
| Mechanism/theory source | Supports interpretation | Discussion |
| Exemplar paper | Teaches structure and evidence sequencing | Blueprint only |

## Exemplar decomposition

For each exemplar, extract only reusable structure:

- central motivation;
- section sequence;
- result sequence;
- figure function sequence;
- baseline/control logic;
- limitation strategy;
- citation function clusters.

Do not extract copyable text.

## Gap mapping

A gap is real only if it satisfies all three conditions:

1. existing work does not already solve it;
2. the user can generate evidence relevant to it;
3. the resulting claim matters to a reader.
